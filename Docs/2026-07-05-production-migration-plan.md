# あしカラダ 本実装移行プラン（作り置き・軽量版）

作成: 2026-07-05 / 現状: A案（上質・高級スパ系）確定・下層9ページ完成・モック公開中（mocks-neon.vercel.app）・クライアントレビュー待ち

## 1. クライアントレビュー反映フロー（次セッションの主タスク）
1. 指摘を「文言 / 写真 / 構成」に分類してリスト化
2. 文言・構成の修正は**共通テンプレート側を直してから各ページへ同期**（個別ページ直修正は同期漏れの既知の罠）
3. 修正後 Playwright 一括スクショ（20枚 PASS の既存検証を再実行）

## 2. 本実装の技術方針【推奨あり】

| 方式 | 評価 | 理由 |
|---|---|---|
| **静的HTML + Vercel（現モックを本番昇格）** | ⭐⭐⭐⭐⭐ | 下層9ページ＋共通テンプレが既に完成品質。readdy 依存を切れて表示速度・自由度も上がる |
| readdy 継続更新 | ⭐⭐ | 移行作業ゼロだが、モックで作った品質を readdy 側で再現する二度手間になる |
| Next.js | ⭐ | 店舗サイトに過剰 |

## 3. 移行設計（静的HTML 昇格の場合の手順）
1. **画像の自前ホスト化**: readdy ストレージ参照の画像（ヒーロー・横揉み・アロマ・技術力等）を `/img/` へ取り込み。差し替え候補は実撮影と合わせて判断
2. **ドメイン移行**: 既存 Punycode ドメイン（xn--l8jzb2o0cyjn09v9ed4ox.jp）の DNS を Vercel へ向ける。**readdy 解約はDNS切替→表示確認の後**（順序厳守・空白期間を作らない）
3. noindex 解除は本番ドメイン切替と同時（モック URL 側は noindex 維持のまま残す）
4. 予約導線の実機確認: 両店舗の電話リンク（tel:）と LINE（緑）が実端末で発火するか。CTA 色ルール（予約=gold 統一・LINE=緑）を崩さない

## 4. 注意
- 並列案件（小林建工）とプレビュー tab を混同しない（既知の事故パターン）
- 公開前チェック: NAP 一貫性（店名・住所・電話が Google ビジネスプロフィールと一致）— MEO 効果に直結
